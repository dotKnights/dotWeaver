# Better Auth + Prisma вЂ” Authentication Setup

**Date:** 2026-05-29  
**Project:** dotWeaver (SvelteKit 5 + TypeScript + Tailwind v4)

## Goal

Set up a full authentication system using Better Auth with Prisma as the ORM, GitHub OAuth + email/password as providers, shadcn-svelte for UI, and Superforms + Zod for form validation.

## Stack

| Layer           | Tool                                        |
| --------------- | ------------------------------------------- |
| Auth framework  | Better Auth                                 |
| ORM             | Prisma (Postgres)                           |
| UI components   | shadcn-svelte                               |
| Form validation | Superforms + Zod                            |
| Database        | PostgreSQL (`dotWeaver` db, already exists) |

## Data Flow

### Email/Password Login

1. User submits form в†’ SvelteKit action in `+page.server.ts`
2. `superValidate` runs Zod schema (email + password)
3. On valid: call `auth.api.signInEmail({ body: { email, password } })`
4. On success: `redirect(303, '/dashboard')`
5. On error: `message(form, errorMessage)` в†’ displayed inline via Superforms

### Email/Password Register

1. User submits form в†’ SvelteKit action
2. Zod schema: email + password + confirmPassword (passwords must match)
3. Call `auth.api.signUpEmail({ body: { email, password, name } })`
4. On success: `redirect(303, '/login')` with confirmation message
5. On error: `message(form, errorMessage)` inline

### GitHub OAuth

- Button calls `authClient.signIn.social({ provider: 'github' })` client-side
- Better Auth handles redirect flow automatically via `hooks.server.ts`
- On callback: session created, redirect to `/dashboard`

### Protected Routes

- `(app)/+layout.server.ts` calls `auth.api.getSession({ headers: event.request.headers })`
- If no session в†’ `redirect(303, '/login')`
- Session object passed to child routes via `locals` or layout data

## Database Schema

Generated by `bunx @better-auth/cli generate` вЂ” produces four tables:

- `user` вЂ” id, email, name, emailVerified, image, createdAt, updatedAt
- `session` вЂ” id, userId, token, expiresAt, ipAddress, userAgent, createdAt, updatedAt
- `account` вЂ” id, userId, accountId, providerId, accessToken, refreshToken, etc.
- `verification` вЂ” id, identifier, value, expiresAt, createdAt, updatedAt

## Zod Schemas

```ts
// Login
z.object({ email: z.string().email(), password: z.string().min(8) });

// Register
z.object({
	email: z.string().email(),
	name: z.string().min(2),
	password: z.string().min(8),
	confirmPassword: z.string().min(8)
}).refine((d) => d.password === d.confirmPassword, {
	message: "Passwords don't match",
	path: ['confirmPassword']
});
```

## UI Components (shadcn-svelte)

- `Card`, `CardHeader`, `CardContent` вЂ” page wrapper
- `Input` вЂ” form fields
- `Button` вЂ” submit + GitHub OAuth button
- `Alert` вЂ” inline error/success messages from Superforms `message()`

## Error Handling

| Scenario                        | Handling                                        |
| ------------------------------- | ----------------------------------------------- |
| Wrong password / user not found | `message(form, 'Invalid credentials')` inline   |
| Email already registered        | `message(form, 'Email already in use')` inline  |
| Passwords don't match           | Zod refine error on `confirmPassword`           |
| No session on protected route   | `redirect(303, '/login')`                       |
| GitHub OAuth error              | Better Auth handles, redirects with error param |
